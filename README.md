# Nurse Fatigue Intelligence & Workforce Optimization Platform

An enterprise-grade, hospital-ready solution that integrates wearable IoT devices (ESP32) with a real-time React dashboard, FastAPI backend, machine learning algorithms, and a Retrieval-Augmented Generation (RAG) knowledge assistant.

---

## 🚀 Key Modules
1. **Attendance Dashboard (Module 1)**: Real-time nurse coverage status, auto-presence via wearable connectivity, visual presence heatmaps, and shift distributions.
2. **Fatigue Monitoring (Module 2)**: Dynamic line charts for Heart Rate, HRV, GSR (Stress Index), Skin Temp, and a composite Fatigue Index.
3. **Smart Replacement Engine (Module 3)**: Automatic lookup of standby qualified nurses with low fatigue (< 40) when active staff experience exhaustion (> 75).
4. **Automatic Shift Optimization (Module 4)**: Roster modification ledger that automatically shortens fatigued shifts and generates new substitute shifts.
5. **RAG Knowledge Assistant (Module 5)**: Offline LangChain policy bot answering workforce guidelines and explaining roster interventions with grounded citations.
6. **Machine Learning Analytics (Module 6)**: SHAP explainable AI coefficients and accuracy indices (XGBoost, Random Forest, SVM, LSTM).
7. **Alert Management (Module 7)**: Triaged warnings (Low, High, Critical) logged across Dashboard, console broadcasts, and SMS/Email dispatch simulations.
8. **Admin Panel (Module 8)**: Roster CRUD inputs, device pairing grids, system audit trails, and threshold calibrations.

## ⚙️ Running Locally (Development Mode)

### Prerequisites
- **Python 3.10+**
- **Node.js 18+ & npm**

### 1. Launch FastAPI Backend
1. Open a terminal and navigate to the `backend/` directory:
   ```bash
   cd backend
   ```
2. Create a virtual environment and activate it:
   ```bash
   python -m venv venv
   # On Windows:
   .\venv\Scripts\activate
   # On macOS/Linux:
   source venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Start the FastAPI server using Uvicorn:
   ```bash
   uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
   ```
   *Note: On initial startup, the database is automatically created at `backend/data/fatigue_platform.db` and seeded with 50 nurses, devices, and 100,000+ historical records. This process might take 5-15 seconds to complete.*

### 2. Launch React Frontend
1. Open a separate terminal and navigate to the `frontend/` directory:
   ```bash
   cd frontend
   ```
2. Install npm modules:
   ```bash
   npm install
   ```
3. Launch the Vite development server:
   ```bash
   npm run dev
   ```
4. Open your web browser and navigate to **`http://localhost:3000`**.
5. Use the following credentials to sign in:
   - **Username**: `admin`
   - **Password**: `admin123`

## 🐳 Running with Docker (Containerized Mode)

To run the entire system inside Docker, execute the following command in the root project folder:
```bash
docker-compose up --build
```
Once initialized, access the React portal at **`http://localhost:3000`** and the Swagger API documentation at **`http://localhost:8000/docs`**.

## 🔌 Wearable Device Ingestion (ESP32 Integration)
- The ESP32 gathers metrics from the **MAX30102**, **GSR**, **MLX90614**, and **MPU6050** sensors.
- It formats telemetry into a JSON payload and publishes it over MQTT to:
  `hospital/sensors/<device_mac>`
- Refer to [wiring_guide.md](file:///C:/Users/DELL/Desktop/IOT_intern_final_projectn/hardware/schematics/wiring_guide.md) for full schematic pinouts and Arduino setup.
- If the hardware is not connected, the backend automatically runs an active simulation loop, drifting nurse telemetries every 2.5 seconds to demonstrate real-time updates and trigger alarms dynamically.
